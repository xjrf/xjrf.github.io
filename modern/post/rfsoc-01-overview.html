RFSoC架构


Zynq UltraScale+ RFSoC
├── PS
│   ├── Quad-core Arm Cortex-A53
│   └── Dual-core Arm Cortex-R5
├── PL
│   └── UltraScale+ FPGA fabric
└── RF Data Converters
    ├── RF-ADC: 4-16个接收通道
    └── RF-DAC: 4-16个发射通道

RF数据转换器包含DDC和DUC，具有可控的插值和抽取率、数控振荡器（NCO）和复数混频器。DDC和DUC支持多频段操作。

Tile组织结构

RF-ADC Tile分为Dual和Quad两种类型。两者的主要区别在于采样率和交织因子：Dual Tile的采样率是Quad Tile的两倍，交织因子也相应更高（8倍/4倍）。

Tile类型

Tile类型转换器数量支持器件说明
Quad RF-ADC4Gen 1/2/3/DFE每个RF-ADC的交织因子为4
Dual RF-ADC2Gen 1/3/DFE每个RF-ADC的交织因子为8，采样率是Quad RF-ADC的两倍
Quad RF-DAC4Gen 1/2/3/DFE每个RF-DAC有1个专用DUC
Dual RF-DAC2Gen 3/DFE每个RF-DAC有2个专用DUC，支持双频段应用

注意：

  • 对于RF-ADC：在 Dual Tile的 ADC中，物理通道0和1对应模拟输入VinX_01和VinX_23。当Tile配置为处理I/Q信号时，必须成对使用（即2个ADC分别处理I路和Q路。因此，这类ADC不支持独立的I/Q输入，因为它们必须成对工作)。
  • 对于RF-DAC：同理，在Dual Tile的DAC中，输出VoutX0和VoutX2对应两个物理通道。要实现I/Q输出，必须同时使用这两个DAC。因此，这类Tile也不支持独立的I/Q输出。
  • RFSoC的IQ模式：RFSoC的DDC和DUC成对工作。也就是说，必须把Tile中的两个转换器（如ADC0和ADC1）作为一个单元，分别处理I路和Q路数据，实现复数变频。所以，即使每个Tile有两个转换器，它们也是捆绑使用的，无法为每个通道提供独立的I/Q接口。而在Quad Tile中，可以形成多个I/Q通道对，配置上会更加灵活。

Tile结构

Quad RF-ADC Tile：


VinX3 → ADC → Data Path → mX3_axis
VinX2 → ADC → Data Path → mX2_axis
VinX1 → ADC → Data Path → mX1_axis
VinX0 → ADC → Data Path → mX0_axis

Dual RF-ADC Tile：


VinX_23 → ADC → Data Path (I/Q) → mX3_axis/mX2_axis
VinX_01 → ADC → Data Path (I/Q) → mX1_axis/mX0_axis

Quad RF-DAC Tile：


sX3_axis → Data Path → DAC → VoutX3
sX2_axis → Data Path → DAC → VoutX2
sX1_axis → Data Path → DAC → VoutX1
sX0_axis → Data Path → DAC → VoutX0

Dual RF-DAC Tile：


sX3_axis/sX2_axis → Data Path (dual DUC) → DAC → VoutX2
sX1_axis/sX0_axis → Data Path (dual DUC) → DAC → VoutX0

代次划分

代次器件型号特性
Gen 1XCZU2xDR基础功能
Gen 2XCZU39DR仅Quad RF-ADC tile
Gen 3XCZU4xDR14位ADC，扩展的抽取/插值率

Gen 3/DFE特有的功能在文档中会明确标注。

Sub-ADC和交织因子

AMD采用时间交织技术实现RF-ADC。Dual RF-ADC tile中的每个RF-ADC由8个sub-ADC组成，Quad RF-ADC tile中的每个RF-ADC由4个sub-ADC组成。

外部时钟输入

并非所有tile都有外部时钟输入：

  • Gen 1/Gen 2：所有tile都有外部时钟输入
  • Gen 3：所有RF-ADC tile有外部时钟输入。对于Quad RF-DAC tile，所有RF-DAC tile有外部时钟输入。如果只有Dual RF-DAC tile，奇数RF-DAC tile（Tile 1和3）没有外部时钟输入
  • DFE和XCZU42DR：底部RF-ADC tile（RF-ADC Tile 0）没有外部时钟输入。其他所有tile有外部时钟输入

没有外部时钟输入的tile需要配置为从器件上的另一个tile接收时钟。在Gen 3/DFE中，任何没有外部时钟输入的tile需要从内部分配网络获取时钟。

额外说明

API和寄存器访问

运行时需通过RFdc驱动API来查询状态和配置RF Data Converter。
